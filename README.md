# ALT Linux Lab Setup (ISP / SRV / CLI)

Шпаргалка по настройке лабораторного стенда на ALT Linux: **маршрутизатор (ISP)**, **сервер (SRV)** и **клиент (CLI)**.  
Внутри — пошаговые команды для сети, DHCP, NAT, RAID5, NFS, SSH и Samba AD DC.

Главный гайд: **[SETUP.md](SETUP.md)**

---

## Что в гайде

- Топология и интерфейсы (ISP/SRV/CLI)
- ISP: статический LAN + IPv4 forwarding
- ISP: DHCP-сервер (включая фиксированный IP для SRV по MAC)
- ISP: NAT через iptables
- SRV: DHCP-интерфейс
- SRV: RAID5 (mdadm) + ФС + fstab
- SRV: NFS-сервер + экспорт
- CLI: монтирование NFS
- SSH: отдельные порты, ключи, алиасы, запрет паролей
- Samba AD DC: домен `ilove.sa`, пользователи/группы, join клиента
- Дополнительно: правила FORWARD (если нужно ограничивать трафик)

---

## Быстрый старт

1) Открой **SETUP.md** и иди по шагам сверху вниз:
   - сначала **ISP** (LAN → forwarding → DHCP → NAT)
   - затем **SRV** (DHCP → RAID5 → NFS → SSH → Samba DC)
   - потом **CLI** (NFS mount → SSH config → join в домен)

2) Если делаешь SSH “как в шпоре”:
   - ISP: порт **2222**, SRV: порт **2223**, только ключи
   - на CLI — алиасы `ISP` и `srv` в `~/.ssh/config`

---

## Требования и допущения

- Используются роли машин: **ISP / SRV / CLI**
- Подсеть: `10.0.128.0/24`, ISP LAN: `10.0.128.1/24`, SRV фиксируется как `10.0.128.2`
- Имена интерфейсов в примере: `enp0s3`, `enp0s8` (смотри “Топология и интерфейсы”)
